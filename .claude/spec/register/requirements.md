# 需求：用户注册功能完善

## 业务场景
新用户进行注册时，需要输入用户名，密码，确认密码，邮箱，邮箱验证码，后端验证无误后返回注册成功

## 验收标准
- 用户的密码和确认密码不相同时提示“两次密码必须相同”
- 密码长度不得小于10位，小于10位前端会提示
- 用户输入的任何数据中包含空格时提示“不可以有空格字符”
- 用户输入错误的邮箱格式时提示“请输入正确的邮箱”
- 用户发送验证码后60秒内无法再次点击发送
- 用户如果用户名重复时，点击注册后会根据后端的返回提示“该用户名已存在”
- 用户注册失败提示后端返回的内容
- 用户注册成功后跳转至登录界面
